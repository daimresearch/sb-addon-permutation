Permutation-Table is a storybook addon that makes it easy to see different looks for your components.
On this demo page, you can see how to use the Permutation-Table Addon.
It's fully compatible with popular front-end component libraries and can be applied to components of your own design.

Explore the stories in the sidebar to learn how to use the Permutation Addon.
Check out the code in the `stories` directory to see how the addon works.
We recommend make a UI with [**Component Driven**](https://componentdriven.org) approach.

## Get started

_Disclaimer: This addon only supports Typescript projects with React and is designed for Storybook 7. It may not work properly in other environments._

0. Perequisites

- `node>=16`
- `yarn` or `npm`
- a working project with:
  - react >= 16.8

1. Installation

```
npm install --save-dev sb-addon-permutation-table

```

```
yarn add -D sb-addon-permutation-table
```

2. Resister the addon in `.stories/main.ts`

```typescript
const config = {
  // ...
  addons: ["sb-addon-permutation-table"],
};
```

3.(optional) Define config for your component in `.stories/your-component.stories.tsx`. Following by type `PermutationMeta` will make this step easy

```typescript
// stories/Component.stories.(ts|tsx)

import React from "react";
import { PermutationMeta } from "sb-addon-permutation-table"; // Not necessary, but it will help you to define the type of meta
import YourComponent from "YourComponent";

const meta: PermutationMeta<typeof YourComponent> = {
  //...
  parameters: {
    permutation: {
      componentName: "Takahashi", // Now "Takahashi" will be shown in the panel, regardless of the component name
      importPath: "@yourLib/yourComponent", // The value copied when you click the "Copy import" button
      children: "<div>Chef of the diamond city</div>", // A value passed to children
      deactivate: ["foo", "bar"], // Deactivate property foo, bar
    },
  },
};
```

4. That's it! Now you can see the permutation table in your storybook.
